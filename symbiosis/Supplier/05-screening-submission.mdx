## Stage 2 Requirements

### What You Need

For Screening Phase 1, prepare:

1. **Questionnaire** - Detailed project information across categories:
   - Methodology eligibility
   - Additionality
   - Permanence
   - Co-benefits

2. **KML file** - Project boundary

3. **Supporting documents** - As specified in the questionnaire:
   - Project Design Document (PDD)
   - Financial model
   - Team CVs
   - Other supporting documentation

4. **Pricing information**

## Questionnaire Completion

### Questionnaire Structure

Questions are organised into:
- **Groups** - Logical groupings within the stage
- **Questions** - Individual data points

### Question Types

| Type | Description | Example |
|------|-------------|---------|
| **Text** | Free text entry | Project description |
| **Number** | Numeric input | Expected volume |
| **Select** | Single choice | Methodology type |
| **Multi-select** | Multiple choices | SDG contributions |
| **Date** | Date selection | Expected registration date |
| **File** | Document upload | PDD, KML file |
| **Table** | Structured data entry | Pricing by vintage |

### Required vs Optional

- **Required questions** marked with asterisk (*)
- Must complete all required questions to submit
- Optional questions can be skipped but completing them strengthens your submission

### Conditional Questions

Some questions appear based on previous answers:
- Answer a trigger question
- Additional questions may appear
- Or questions may become hidden

## Document Upload

### Supported File Types

| Document Type | Accepted Formats |
|---------------|------------------|
| Text documents | `.pdf`, `.docx` |
| Spreadsheets | `.xlsx`, `.csv` |
| Images | `.png`, `.jpg`, `.jpeg` |
| Geospatial | `.kml`, `.kmz` |

### File Size Limits

- Individual file: 25 MB maximum
- Total submission: 100 MB maximum

### Naming Conventions

Use clear, descriptive file names:
- **Good:** `ProjectName_PDD_v1.pdf`
- **Avoid:** `Document1.pdf`, `Final_FINAL.docx`

### Uploading Files

1. Navigate to the document question
2. Click **'Upload'** or **'Choose File'**
3. Select file from your device
4. Wait for upload to complete
5. Verify upload successful

## Review and Submit

### Pre-submission Review

Before submitting:
1. Click **'Review Submission'**
2. System checks all required fields
3. View summary of your responses
4. Address any validation errors

### Validation Checks

The platform automatically checks:

| Check | Description |
|-------|-------------|
| Required fields | All mandatory fields are completed |
| File formats | Uploaded files match accepted formats |
| KML validity | Boundary geometry is valid |
| Data consistency | Cross-field validation |

### Validation Errors

If validation fails:

| Error Type | Description |
|------------|-------------|
| **Missing Required** | Required question not answered |
| **Invalid Format** | Answer doesn't match expected format |
| **Incomplete Upload** | Document not fully uploaded |

### Submitting

1. Ensure all validations pass
2. Review submission summary
3. Check acceptance checkbox (if required)
4. Click **'Submit'**
5. Confirmation displayed

### After Submission

- Confirmation email sent
- Submission locked for editing
- Status visible in your dashboard
- Review begins according to the rolling schedule

---

## Data Locking

### How Locking Works

- Once you submit, your data is **locked** for review
- You cannot edit locked submissions
- If changes are needed, Symbiosis will request them via the platform

### Version History

- All submissions are versioned
- Previous versions are retained
- You can view your submission history

### Requesting Updates

If you need to update a submitted proposal:

1. Wait for a request from Symbiosis, or
2. Contact support to request an update

*Note: Updates are only possible before review is complete.*

## Common Issues

### Cannot Submit

- Check all required fields are completed
- Verify all file uploads succeeded
- Review validation error messages

### Upload Failing

- Check file size is within limits
- Verify file format is supported
- Try a different browser or clear cache

### Need to Edit After Submit

- Contact the buyer through the platform
- Request unlock for specific fields
- If approved, fields become editable
- Re-submit when changes complete
